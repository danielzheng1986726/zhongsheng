## 一、项目概况

「众声 Voices」— 知乎讨论理解工具。用户选择热榜话题或粘贴链接，AI 分析回答、提炼对立观点，生成法庭风格辩论剧本，最后由 Second Me 分身出庭旁听。

- **线上地址**：https://zhongsheng.ai-builders.space
- **GitHub**：https://github.com/danielzheng1986726/zhongsheng
- **部署**：AI Builder Space（`POST /backend/v1/deployments` 手动触发，自动部署不可靠）

## 二、关键决策（本次会话）

### 2.1 广场动态 + 融合流

| 决策 | 结论 |
|------|------|
| 旁听席 → 广场 | 旁听席改为「广场动态」，占主要 feeds 区域 |
| 评论来源标识 | 每条评论标注「真人」或「AI 分身」（`source: human/agent`） |
| 话题选择器 | **可选**。默认「💬 自由发言」，无辩论时也可发评论，打破鸡生蛋问题 |
| 自由评论 | 新增 `POST /api/plaza/free-comment`，评论不绑定辩论，存 `plaza.json` |
| 热榜 + 广场融合 | 热榜条目与广场评论混排在同一 feeds 流（`/api/unified-feed`） |

### 2.2 布局设计方向

- **融合流方案**：热榜话题卡片 + 广场评论穿插在同一个 feed，参考知乎推荐/关注流
- **视觉区分**：热榜用大卡片（带热度），评论用紧凑样式
- **Web vs 移动**：可做响应式——桌面端中间 feed + 右侧热榜，移动端单列

### 2.3 知乎分享链接解析

- 用户复制知乎「分享」会得到 `标题\nURL` 格式
- 前端解析：提取 title 和 url，分别传给后端，避免服务端爬知乎标题（403 反爬）

## 三、当前实现状态

### 3.1 主要 API

| 端点 | 说明 |
|------|------|
| `GET /api/plaza` | 广场聚合流（辩论评论 + 自由评论 + auditorium 反馈） |
| `GET /api/unified-feed` | 融合流：热榜 + 辩论回放 + 广场评论，交错排序 |
| `POST /api/plaza/free-comment` | 自由发言（Body: `{text, nickname?, source?}`） |
| `POST /api/plaza/agent-comment` | 分身评论（需登录，Body: `{debate_id}`） |
| `POST /api/debate/{id}/comment` | 绑定辩论的评论（Body: `{text, nickname?, source?}`） |

### 3.2 评论数据结构

```json
{
  "text": "...",
  "nickname": "匿名旁听",
  "source": "human | agent",
  "debate_topic": "辩论话题（自由发言为空）",
  "debate_id": "辩论 ID（自由发言为空）",
  "ts": 1234567890
}
```

### 3.3 持久化文件

| 文件 | 内容 |
|------|------|
| `zhihu_cache/debates.json` | 辩论索引（不含 script/chars） |
| `zhihu_cache/debate_{id}.json` | 单场辩论回放数据 |
| `zhihu_cache/plaza.json` | 广场自由评论 |

### 3.4 前端结构（`static/index.html`）

- **Unified Feed**：`feedSection` + `feedList`，由 `loadFeed()` 拉取 `/api/unified-feed`
- **Compose 区**：`plazaInput` + `plazaTopicSelect` + `plazaSendBtn` + `plazaAgentBtn`
- 发表 / 分身评论后调用 `loadFeed()` 刷新流

## 四、已修复的 Bug

### 4.1 广场评论不即时刷新

**原因**：原先 `plazaEmpty` 放在 `plazaFeed` 内，第一次 `innerHTML` 渲染会把它覆盖，后续 `$('plazaEmpty')` 为 null，抛错被 catch 静默，导致 `el.innerHTML` 不执行。

**修复**：将 `plazaEmpty` 移为 `plazaFeed` 的兄弟元素，且增加 null 检查。  
（若已切换为 unified-feed 布局，该问题可能已不存在，因 `feedList` 直接整体替换 innerHTML。）

### 4.2 知乎链接解析 403

**原因**：服务端 `get_question_title` 爬知乎页面被反爬。

**修复**：前端解析分享格式 `title\nURL`，将 title 和 url 分开传给后端。
