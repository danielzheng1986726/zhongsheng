# 众声 Post-Hackathon 行动计划

## 第一阶段：Build in Public（第 1-2 周）

**目标：建立个人品牌 + 开发者社区影响力**

### Week 1：写 3 篇内容

**帖子 1：黑客松复盘（小红书 + 知乎）**
- 标题方向：「一个 HR 用 AI 写代码参加黑客松，没进决赛但被知乎市场部转发到了内部群」
- 内容要素：
  - 背景：零编码经验，用 Vibe Coding 一周做出完整产品
  - 数据：108 人用户群但只有 17 个 OAuth（教训：Day 1 不设登录墙 = 白忙）
  - 转折：评委反馈极其正面（"马上转到内部群"）
  - 金句：「他们以为自己在反对彼此，其实在反对同一堵墙」
  - 结尾：附产品链接 + 二维码
- 不要写成总结报告，写成**故事**

**帖子 2：技术决策故事（知乎 + X/Twitter）**
- 标题方向：「我用知乎圈子当数据库，省了一个 Postgres」
- 内容要素：
  - 为什么没用传统数据库（黑客松时间紧 + 容器重启数据丢失）
  - 知乎圈子发布 API 作为持久化层的巧妙设计
  - 后来加了 Turso 但双轨运行的架构
  - 配架构图
- 这种「反直觉技术决策」在开发者社区传播效果好

**帖子 3：AI 时代 GTM 洞察（X/Twitter 英文线程）**
- 标题方向：「I built a full product in 7 days. Got 17 users. Here's what I learned about GTM vs Code.」
- 内容要素：
  - Code is middleware now（引用 Karpathy/YC/Sequoia）
  - 黑客松第一名是全球化 + Day 1 登录墙，不是最好的产品
  - GTM > Code 的具体数据对比
- 英文线程可以触达全球独立开发者社区

### Week 2：建立发布节奏

- **频率**：每周 3 次（周一/周三/周五）
- **平台**：小红书（中文主阵地）+ X/Twitter（英文扩展）
- **内容轮换**：
  - 周一：产品更新 / 新功能演示
  - 周三：洞察 / 思考 / 行业观点
  - 周五：数据 / 收入 / 用户反馈
- **互动**：每天 15 分钟回复评论

## 第三阶段：体验强化（第 3-4 周）

**目标：把众声的法庭体验做到「Cursor 级别」的不可替代**

### 已有的体验优势
- 法庭音效（法槌、反对、打字机）
- "我反对！"按钮 + 屏幕震动
- 刘看山审判长 IP
- 共识揭示的仪式感

### 需要强化的

**任务 1：压缩 gavel.png（保留特色，优化格式）**
- 2.1 MB → WebP 格式 < 200 KB
- 不改设计，只换格式

**任务 2：首次加载体验优化**
- 目标：Time to Wow < 10 秒
- 加载动画 → 第一声法槌落下 → 用户立刻被吸引
- 预加载音效文件

**任务 3：产品多语言（英文版）**
- 全球化是最大获客杠杆（Linked 的 1007 agent 主要来自海外）
- 前端文案翻译（index.html 里的中文 → 中英切换）
- MCP 工具描述加英文版
- 这一步可以用 AI 快速完成

**任务 4：移动端体验打磨**
- 法庭动画在手机上的流畅度
- 搜索框下拉在小屏幕的适配
- 分享海报在手机上的生成质量

## 第四阶段：Agent 分发飞轮（第 4-6 周）

**目标：让 Agent 成为持续获客渠道**

### 飞轮机制
```
Build in Public 帖子
    → 人类开发者发现众声
    → 安装 MCP Server 到自己的 Agent
    → Agent 在对话中使用众声工具
    → 更多 Agent 的使用数据
    → 众声在 MCP 市场排名上升
    → 更多 Agent 自动发现并使用
    → 回到 Build in Public 分享增长数据
```

### 具体行动

**任务 1：监控 Agent 使用数据**
- 在 MCP 端点加日志：哪些工具被调用最多、频率、来源
- 每周分享数据（Build in Public 素材）

**任务 2：优化 Agent 体验**
- Agent 调用 `zhongsheng_search("躺平")` 后返回的结果要足够好
- 返回格式要对 Agent 友好（结构化 JSON，不是长文本）
- Agent 不需要"好看"，需要"好用"

**任务 3：建立反馈循环**
- 如果有 Agent 频繁调用某个工具，说明那个能力最有价值
- 根据 Agent 使用数据调整产品方向

**任务 4：探索 Agent Commerce**
- 关注 Google UCP 协议进展
- 如果众声能提供付费的深度分析（比如品牌舆情），Agent 可以代理购买
- 这是长期方向，现在先关注

## 不要做的事情（Anti-patterns）

1. **不要继续优化代码而不发布内容** — 代码是中间件，分发才是杠杆
2. **不要等产品"完美"再开始 Build in Public** — 不完美就是最好的内容
3. **不要只在中文社区推广** — 全球化是 10 倍杠杆
4. **不要把 Build in Public 变成"内容创作者"** — 你是产品构建者，内容是副产品
5. **不要忽视 Agent 可发现性** — 这是下一个 SEO，早做早受益
