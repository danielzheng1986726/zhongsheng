# 众声 设计笔记

## 加载页设计（优先级高）

上一届教训：人格匹配报告加载近 1 分钟，只显示「匹配中」，用户大量流失。

原则：**加载页本身就是内容，不是等待。**

参考游戏加载页的做法：
- 画面必须极其漂亮或极其有趣
- 可以展示「正在分析中」的动态过程（比如 Agent 头像逐个亮起、讨论内容流过屏幕）
- 让等待变成期待，而不是忍耐

## 动态内容 + 永不重复的剧本

- 如果知乎提供开放 API，可自动拉取热帖，每次生成全新的辩论
- 剧情跟着时事走，不会重复——这是游戏化的核心
- 用户也可以自己贴链接选择讨论话题
- Agent 是「世界舞台上的演员」，人类是「平行宇宙的观察者」
- 当前版本：用户观赏剧情，暂不与 Agent 直接交互（未来可扩展）

## 表情包生成策略

必须用 AI Builder Space 的 image generation API 生成，不用外部工具。原因：

1. **每次生成都不同** — 这是 feature 不是 bug
2. **抓核心特征，允许走样** — 比如「葛优瘫」的核是：秃顶男人 + 破衬衫 + 90 年代沙发 + 半躺姿势。AI 不会画出完全一样的葛优，但会画出「AI 理解的葛优瘫」，这个走样本身就是笑点
3. **保证动态性** — 同一个阵营每次生成的角色形象略有不同，增加新鲜感和重玩性
